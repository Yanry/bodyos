## Implementation Log

### 理解要点

1. **文案中心化**:
   - 所有 UI 层面的文本描述应统一从此库调用，确保全 App 语气的一致性。
   - 语气强调「日常化」和「具象化」，这与 F2 (Camera Preparation) 和 T2 (Problem Description) 的目标高度一致。

2. **多模态匹配**:
   - `距离提示` 需要区分前置和后置两种物理情况，逻辑上需根据相机状态动态切换文本。
   - `入框状态` 是实时的逻辑反馈，需要极高的一致性，避免歧义。

3. **语气原则落地**:
   - 避免「屈髋」、「外旋」等术语，转而使用库中的日常化表达。
   - 强调「为什么」，提升排错指令的说服力。

### 实现计划

1. **文案常量定义**:
   - 在 `src/constants/ui-copy.ts` 中根据分类（Shooting, Recording, Buttons）导出对象。
2. **国际化/多语言架构** (虽然目前仅中文):
   - 即使是单语言，也建议采用类似 `t('system.buttons.retry')` 的模式，方便后续微调文案而无需改动业务逻辑。
3. **动态提示逻辑**:
   - 实现 `getDistanceHint(cameraType)` 等工具函数。

### 待澄清问题

无 - 文案清晰，原则明确。

## 语气原则

- 用日常语言，避免技术术语
- 具体的动作意象，不要抽象指令
- 强调「为什么」和「有什么影响」
- 具身认知：联系身体感受
